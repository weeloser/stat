# Командный центр

Командный центр позволяет управлять сессиями, сделками, статистикой и внешним видом без перехода по формам. Он работает только с командами из фиксированного списка и не исполняет JavaScript, HTML или системные команды.

## Как открыть и запустить команду

- Нажмите кнопку командного центра справа сверху или `Ctrl+K` (`Cmd+K` на macOS).
- `Enter` запускает предварительный просмотр команды.
- `Shift+Enter` добавляет новую строку в пакет.
- `Tab` принимает выбранную подсказку.
- `↑` и `↓` выбирают подсказки; `Ctrl+↑` / `Ctrl+↓` листают историю.
- `Esc` закрывает окно.

Команда, которая меняет сохранённые данные, сначала показывает план. Данные изменятся только после нажатия **«Применить»**. Открытие сессии и служебные команды без изменений применяются сразу. Удаление и очистка дополнительно требуют флаг `--confirm`; для особо больших удалений появится контрольная галочка.

## Быстрый старт

```text
сессии список
сессия создать "Скальпинг — июль" баланс=10000
сессия открыть "Скальпинг — июль"
сделка добавить BTC long 60000 -> 61800 x2
сделка изменить #1 тег=scalp заметка="Вход по плану"
статистика показать current
```

Английские и русские слова можно смешивать:

```text
trade add ETH short entry=3500 exit=3400 leverage=3 tag=scalp
настройка set тема=light акцент=#ff8a74
```

## Общий синтаксис

```text
<объект> <действие> [цель] [ключ=значение ...] [флаг]
```

Поддерживаемые объекты:

- `сессия`, `сессии`, `session`, `sessions`;
- `сделка`, `сделки`, `trade`, `trades`;
- `статистика`, `итоги`, `stats`, `statistics`;
- `настройка`, `настройки`, `config`, `settings`.

Команды и значения `long`/`short` не зависят от регистра. Тикер приводится к верхнему регистру. Текст с пробелами заключайте в одинарные или двойные кавычки:

```text
сессия создать "Фьючерсы BTC — вечер"
сделка изменить #1 заметка="План: дождаться ретеста"
```

Внутри кавычек можно экранировать символ обратной косой чертой. `\n` и `\t` превращаются в перевод строки и табуляцию. Точка с запятой внутри кавычек не разбивает пакет.

Важно:

- комментарии через `#` не поддерживаются: `#1` используется как номер сделки;
- повтор одного ключа не считается ошибкой — используется последнее значение;
- проценты вводятся числом без `%`: `pnl=3.5`, а не `pnl=3.5%`;
- поддерживаются `сегодня`/`today`, `вчера`/`yesterday` и даты `YYYY-MM-DD`;
- неизвестный ключ вызывает ошибку; неизвестные флаги не следует использовать.

## Сессии

### Показать список

```text
сессии список
session list
```

Таблица показывает название, количество сделок, P&L, winrate и ID каждой сессии.

### Создать

```text
сессия создать "Основной счёт"
сессия создать имя="Тест риска" баланс=10000
```

Название обязательно. `баланс` — справочное поле сессии.

### Открыть

```text
сессия открыть current
сессия открыть "Основной счёт"
сессия открыть s-1720000000000
```

Селектором может быть `current`/`текущая`/`.`, полный ID, точное название или единственное частичное совпадение по названию/ID. Неоднозначное совпадение отклоняется.

### Переименовать и изменить баланс

```text
сессия переименовать current "Основной счёт — Q3"
сессия изменить current имя="Q3" баланс=12000
```

При `изменить` новое имя всё равно обязательно; отдельно изменить только баланс этой командой нельзя.

### Копировать

```text
сессия копировать current
сессия копировать current имя="Q3 — копия"
```

Копируются все сделки, но с новыми уникальными ID. Если имя не задано, добавляется `— копия`.

### Удалить

```text
сессия удалить "Старый тест" --confirm
сессия удалить current --confirm
сессия удалить все --confirm
```

`все`/`all`/`*` удаляет все сессии и требует дополнительную галочку в интерфейсе. Любое удаление выполняется только после preview и нажатия «Применить».

## Сделки

### Добавить одну сделку

Короткая запись:

```text
сделка добавить BTC long 60000 -> 61800 x2
trade add ETH short 3500 -> 3400 x3
```

Запись через ключи:

```text
сделка добавить coin=BTC dir=long вход=60000 выход=61800 плечо=2 дата=2026-07-16 тег=scalp заметка="По плану"
```

Для новой сделки обязательны:

- тикер;
- положительная цена входа;
- `выход`, `pnl` или `фиксации`.

По умолчанию используются `long`, плечо `1`, доля выхода `100` и текущая дата. Открытую сделку без выхода командный центр создать не может.

Поддерживаемые ключи сделки:

| Поле | Алиасы | Формат |
|---|---|---|
| Тикер | `coin`, `symbol`, `ticker`, `тикер`, `монета` | до 32 букв/цифр и символов `._/-` |
| Направление | `dir`, `direction`, `направление` | `long`, `short`, `лонг`, `шорт`, `buy`, `sell` |
| Вход | `entry`, `вход` | положительное число |
| Выход | `exit`, `выход` | положительное число |
| Плечо | `margin`, `leverage`, `x`, `плечо`, `маржа` | число больше нуля, максимум 1000 |
| Доля | `pct`, `percent`, `share`, `доля`, `процент` | число больше 0 и не больше 100, без знака `%` |
| P&L | `pnl`, `result`, `результат`, `прибыль` | число без знака `%`; создаётся синтетический выход |
| Дата | `date`, `дата`, `time` | `YYYY-MM-DD`, `сегодня`, `вчера` |
| Тег | `tag`, `тег` | до 40 символов |
| Заметка | `note`, `comment`, `заметка`, `комментарий` | до 500 символов |
| Фиксации | `fixes`, `exits`, `фиксации`, `выходы` | `цена@доля,цена@доля` |
| Сессия | `session`, `сессия`, `in`, `в` | селектор сессии |

### Несколько фиксаций

Используйте запятую между фиксациями и `@` между ценой и долей:

```text
сделка добавить SOL long вход=140 плечо=2 фиксации=145@25,150@50,160@25
```

Доля одной фиксации по умолчанию равна 100. Сумма долей может быть меньше 100, но не больше 100. Максимум — 100 фиксаций.

### Добавить сделку по известному P&L

```text
сделка добавить XRP long вход=0.52 плечо=5 pnl=7.5
сделка добавить BTC short вход=61000 x=2 pnl=-3.25
```

Командный центр вычислит один синтетический выход. Если одновременно передать `выход` и `pnl`, используется явно заданный выход. Если передать `фиксации`, они имеют приоритет над `выход` и `pnl`.

### Добавить 2–3 и больше сделок разом

Разделяйте команды `;` или переносом строки:

```text
trade add BTC long 60000 -> 61800 x2;
trade add ETH short 3500 -> 3400 x3;
trade add SOL long 142 -> 148 x2
```

После полной команды `trade add` можно продолжить следующими сделками без повторения слов `trade add`:

```text
trade add BTC long 60000 -> 61800 x2; ETH short 3500 -> 3400 x3; SOL long 142 -> 148 x2
```

Знак `+` как сокращение не поддерживается. В одном запуске действует предел 60 команд (внутренний защитный предел новых сделок — 250).

### Показать список

```text
сделки список
сделки список лимит=100 сортировка=profit
сделки список сессия="Основной счёт" сортировка=oldest
```

`лимит` принимает значения 1–500. Сортировка списка: `newest` (по умолчанию), `oldest` или `profit`.

### Как выбрать сделку

Поддерживаются:

- `#1`, `#2` — позиция сделки в исходном порядке сессии;
- полный ID или единственное частичное совпадение по ID;
- тикер, если такая сделка одна;
- `coin:BTC` или `тикер:BTC` — все сделки с тикером;
- список через запятую: `#1,#3,t-1720000000000`;
- `все`, `all`, `*`.

Селекторы `последняя` и `where`/`где` не поддерживаются.

### Изменить

```text
сделка изменить #1 вход=61250 выход=62000 плечо=3
сделка изменить id=t-1720000000000 тег=review заметка="Проверить"
сделка изменить #2 фиксации=145@50,150@50
```

За одну команду можно изменить только одну сделку. Доступны поля тикера, направления, входа, выхода/P&L/фиксаций, плеча, доли, даты, тега и заметки. Пустые `тег=""` и `заметка=""` очищают эти поля.

### Копировать

```text
сделка копировать #1
сделка копировать #1,#2
сделка копировать coin:BTC
```

Каждая выбранная сделка копируется один раз, получает новый ID и текущую дату. Параметр количества копий не поддерживается.

### Удалить

```text
сделка удалить #1 --confirm
сделка удалить #1,#3 --confirm
сделка удалить coin:BTC --confirm
сделка удалить все --confirm
```

Удаление всегда требует `--confirm`. Удаление всех сделок или 20+ сделок дополнительно требует контрольную галочку.

## Статистика

```text
статистика показать current
статистика показать "Основной счёт"
статистика показать все
```

Показываются: число сделок, P&L, winrate, Win/Loss/BE, средний P&L и Profit Factor.

Очистить сделки, сохранив сессию:

```text
статистика очистить current --confirm
статистика очистить "Основной счёт" --confirm
статистика очистить все --confirm
```

`статистика удалить` действует так же, как `статистика очистить`: удаляются сделки, а не сессия. Создание, открытие и переименование сессий выполняйте через объект `сессия`.

## Настройки

Показать текущие значения:

```text
настройки показать
```

Изменить одно или несколько значений:

```text
настройка установить тема=light акцент=#ff8a74
настройка установить сортировка=profit компактно=true фокус=false
настройка установить толькочтение=true архив=true
настройка установить сортировкасессий=favorites
```

Поддерживаемые настройки:

| Ключ | Значения |
|---|---|
| `theme` / `тема` | `dark`, `light` |
| `sort` / `сортировка` | `newest`, `oldest`, `profit` |
| `accent` / `акцент` / `цвет` | цвет `#RRGGBB`; `акцент=` возвращает стандартный цвет |
| `dense` / `compact` / `компактно` | `true/false`, `да/нет`, `вкл/выкл` |
| `focus` / `фокус` | `true/false`, `да/нет`, `вкл/выкл` |
| `locked` / `readonly` / `толькочтение` | `true/false`, `да/нет`, `вкл/выкл` |
| `archived` / `archive` / `архив` | показывать архивные сессии: `true/false` |
| `sessionsort` / `сортировкасессий` | `newest`, `oldest`, `profit`, `winrate`, `trades`, `name`, `favorites` |

Сбросить только визуальные настройки (данные не затрагиваются):

```text
настройка сбросить
```

Когда `толькочтение=true`, команды изменения сессий, сделок и статистики запрещены. Изменить сами настройки по-прежнему можно:

```text
настройка установить толькочтение=false
```

## Пакеты, preview и безопасность

Пакет состоит из команд, разделённых `;` или переносами строк. Сначала весь пакет разбирается на копии данных. Если любая команда ошибочна, применить пакет нельзя. При сохранении все изменения записываются вместе; при ошибке записи восстанавливается предыдущий снимок.

```text
сессия создать "Нью-Йорк";
сессия открыть "Нью-Йорк";
сделка добавить BTC long 60000 -> 61200 x2;
сделка добавить ETH short вход=3500 pnl=3.5
```

Предпросмотр без возможности применить:

```text
сделка удалить coin:BTC --confirm --dry-run
```

Поддерживаемые флаги:

| Флаг | Назначение |
|---|---|
| `--dry-run`, `--preview`, `--тест` | показать план и отключить кнопку применения |
| `--confirm`, `--подтвердить` | разрешить опасную операцию после preview |

Флаги действуют на весь пакет независимо от того, в какой подкоманде стоят. `-y`, `--json` и `# комментарий` не поддерживаются.

Ограничения одного запуска:

- до 12 000 символов;
- до 60 команд;
- до 800 токенов;
- до 60 команд (в обычном пакете это означает до 60 новых сделок; внутренний защитный предел новых сделок — 250);
- создание новой сессии блокируется после 1000 существующих сессий;
- до 20 000 сделок в одной сессии;
- до 100 фиксаций в одной сделке.

## Служебные команды

```text
помощь
примеры
история
очиститьэкран
```

`помощь` и `примеры` показывают общую встроенную памятку. Тематическая форма вроде `помощь сделка` сейчас не поддерживается. `история` выводит до 20 последних введённых команд, а хранится до 100. `очиститьэкран` очищает только область ответа командного центра.

Команд `отменить`, `повторить`, `экспорт`, `импорт`, `домой`, `режим чтение`, архивирования/избранного сессии и массовых фильтров `где` в текущей версии нет.

## Короткая памятка

```text
сессии список
сессия создать "Название" баланс=10000
сессия открыть "Название"
сделка добавить BTC long 60000 -> 62000 x3
сделка добавить SOL long вход=140 фиксации=145@50,150@50
сделка изменить #1 тег=review
сделка удалить #1 --confirm
статистика показать current
настройка установить акцент=#ff8a74
```

Если команда удаляет данные, сначала добавьте `--dry-run`, проверьте план, затем уберите `--dry-run`, оставьте `--confirm` и примените изменения.
